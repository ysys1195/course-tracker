# AGENTS.md

このリポジトリでは、CourseTracker の公開用ポートフォリオ実装を行う。

## プロダクト方針

- CourseTracker は、技術学習に使う公式 Docs、公式 GitHub、動画、書籍、記事を一元管理する学習ダッシュボード。
- 公開用ポートフォリオとして安全に見せられる題材にする。
- 求人情報、企業情報、選考状況、個人の転職活動ログは扱わない。
- seed データ、スクリーンショット、README、テストデータには架空または公開して問題ない教材情報だけを使う。

## 技術方針

- フレームワーク: Next.js
- 言語: TypeScript
- DB: PostgreSQL
- ORM: Prisma
- 認証: Auth.js
- 本番デプロイ: Vercel Hobby
- 本番DB: Neon Free
- ローカルDB: Docker + PostgreSQL を第一候補にする
- UI は Tailwind CSS を基本候補にする

## UI / UX 方針

- アプリ内の文言は原則として日本語にする。
- 説明文を長く置きすぎず、状態・アイコン・ラベル・カード設計で直感的に伝える。
- README や設計ドキュメントは日本語で書く。
- 空状態、ローディング状態、エラー状態を明示的に設計する。
- モバイル幅でも主要導線が使えるようにする。
- 公開デモでは個人情報や実際の学習履歴を含めない。

## データ設計方針

- 中心モデルは LearningResource とする。
- 学習教材には、公式 Docs、公式 GitHub、動画、書籍、記事、チュートリアルなどの種別を持たせる。
- タグ、学習メモ、学習ログ、学習ロードマップを拡張しやすい形で設計する。
- GitHub リポジトリ URL は教材 URL として保存する。必要になるまで GitHub API 連携は追加しない。
- 画像やPDFなどの重いデータはDBに保存しない。必要な場合は外部URLを保存する。

## セキュリティ / 公開ルール

- `.env` や秘密情報をコミットしない。
- `DATABASE_URL`、Auth.js の secret、OAuth client secret は必ず環境変数で管理する。
- seed データには公開情報または架空データだけを使う。
- 実在サービスの公式URLを使う場合は、個人メモや非公開情報を混ぜない。
- Prisma Studio はローカル確認用途に限定し、公開しない。

## ドキュメント方針

- README には以下を含める。
  - 概要
  - デモURL
  - スクリーンショット
  - 主要機能
  - 技術スタック
  - アーキテクチャ
  - ER図
  - 画面遷移図
  - セットアップ手順
  - 今後の改善
- ER図と画面遷移図は Mermaid を基本にする。
- スクリーンショットはあとから差し替え可能なプレースホルダーで始めてよい。

## 開発運用

- コミットメッセージは日本語で書く。
- Issue、PR、README、設計メモも原則として日本語で書く。
- 変更前に既存ファイルを確認し、ユーザーの変更を巻き戻さない。
- 破壊的な git 操作は行わない。
- 実装では、型安全性、バリデーション、認可、エラーハンドリングを軽視しない。
