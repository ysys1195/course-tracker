# Frontend Skill

この skill は、コンポーネント数の多さよりも、アートディレクション、情報の強弱、余白の取り方、画像の使い方、モーションの質が成果を左右するタスクで使ってください。

目標は、意図が感じられ、上質で、今っぽさのあるUIを仕上げることです。  
基本方針は、ひとつの強いアイデア、印象に残るビジュアル、少ないコピー、厳密な余白設計、そして記憶に残る少数のモーションです。

## 作業の進め方

実装を始める前に、まず次の3つを言語化してください。

- visual thesis: どんな雰囲気・素材感・エネルギーを持たせるかを一文で表す
- content plan: hero / support / detail / final CTA の流れを整理する
- interaction thesis: ページ全体の印象を変える 2〜3 個のモーション方針を決める

各セクションには、1つの役割、1つの主役となるビジュアル、1つの主な学びや行動だけを持たせます。

## 基本方針

- コンポーネントからではなく、まず構成から考える
- 全幅のヒーロー、または画面全体を支える強いビジュアルを優先する
- ブランド名またはプロダクト名を、画面内で最も強い文字要素にする
- コピーは、数秒で流し読みできる短さに抑える
- 枠や装飾を足す前に、余白、整列、サイズ差、トリミング、コントラストで解決する
- 使用するルールは絞る。書体は基本2種類まで、アクセントカラーは基本1色まで
- デフォルトはカードなしのレイアウトにする。必要ならセクション、カラム、区切り、リスト、メディアブロックで構成する
- 最初の画面は、文書ではなくポスターとして扱う

## ランディングページ

基本構成は次の順番を標準とします。

1. Hero: ブランドまたはプロダクト、約束、CTA、そして1つの主役ビジュアル
2. Support: 機能、オファー、または信頼材料を1つ具体的に示す
3. Detail: 世界観、体験の流れ、プロダクトの深み、またはストーリーを見せる
4. Final CTA: 始める、申し込む、訪問する、問い合わせるなどの行動へつなげる

### Hero のルール

- 構成の主役は1つだけにする
- 全幅画像、または画面を支配するビジュアル面を使う
- ブランド訴求型のランディングページでは、ヒーロー自体は画面端まで広げる。ページ全体の共通ガターや囲みコンテナに閉じ込めない。制約をかけるのは内側のテキスト列だけにする
- 優先順位は、ブランド → 見出し → 本文 → CTA
- 初期案として、ヒーローカード、統計帯、ロゴ一覧、ピルの乱立、浮いたダッシュボード風要素は置かない
- 見出しは、デスクトップで 2〜3 行程度、モバイルではひと目で読める長さにする
- テキストは細めのカラムにまとめ、画像の中でも落ち着いた領域に配置する
- 画像上に文字を重ねる場合は、十分なコントラストと押しやすいタップ領域を確保する

画像を消しても最初の画面が成立してしまうなら、その画像は弱すぎます。  
ナビゲーションを消しただけでブランド感が薄れるなら、情報の強弱づけが弱すぎます。

### ビューポートの考え方

- 最初の画面に sticky /
  fixed ヘッダーがある場合、そのヘッダーもヒーローの表示コストに含める
- ヘッダーとヒーローを合わせた内容は、一般的なデスクトップとモバイルの初期表示内に収まるようにする
- `100vh` や `100svh`
  を使う場合は、常時表示UIの高さを差し引くか、ヘッダーを通常フローに積まず重ねて扱う

## アプリUI

アプリでは、Linear のような抑制された方向を標準とします。

- 落ち着いた面の階層
- 強いタイポグラフィと余白
- 少ない色数
- 情報量は多くても読みやすいこと
- 装飾は最小限
- カードは、そのカード自体が操作対象である場合に限って使う

構成は次の軸で整理してください。

- primary workspace
- navigation
- secondary context または inspector
- 行動や状態を示すための、明確なアクセント1つ

避けるもの:

- ダッシュボードカードを敷き詰めたような構成
- 全領域に太い境界線を置くこと
- 日常的な業務UIの背後に装飾的なグラデーションを置くこと
- 複数のアクセントカラーが競合すること
- スキャン性に寄与しない装飾アイコン

意味を失わずにプレーンなレイアウトへ戻せるなら、そのカード表現は不要です。

## 画像の扱い

画像は、見た目の飾りではなく、意味を運ぶものでなければなりません。

- ブランド、店舗、エディトリアル、ライフスタイル系のページでは、少なくとも1枚は強くて実在感のある画像を使う
- 抽象グラデーションや偽物っぽい3Dオブジェクトより、現場感のある写真を優先する
- 文字を載せるなら、明るさや情報量が安定した領域を持つ画像を選ぶ、または適切にトリミングする
- 看板、ロゴ、文字ノイズが多く、UIと競合する画像は使わない
- UIフレーム、カード、分割パネルが最初から描き込まれた画像は使わない
- 複数の瞬間を見せたいなら、1枚のコラージュではなく複数画像で表現する

最初の画面には、実感のあるビジュアルアンカーが必要です。  
装飾的なテクスチャだけでは足りません。

## コピー

- デザインの説明ではなく、プロダクトの言葉で書く
- 意味はまず見出しに持たせる
- 補足文は、基本的に短い一文で十分
- セクションをまたいだ重複表現は削る
- プロンプト文やデザイン解説をそのままUIへ出さない
- 各セクションには、説明する、証明する、深める、転換する、のどれか1つの役割だけを持たせる

30% 削った方が良くなるなら、さらに削ってください。

## プロダクトUI向けの文言

ダッシュボード、管理画面、業務ツール、運用画面では、マーケティング文言ではなくユーティリティ文言を基本にします。

- 約束や世界観より、状況把握、状態表示、次の行動を優先する
- まずは作業画面そのものから始める。KPI、チャート、フィルター、テーブル、状態、タスク文脈を優先する
- 明示的に求められない限り、ヒーローセクションは導入しない
- 見出しは、その領域が何か、または何ができるかを直接示す
- 例: "主要KPI", "プランの状態", "検索指標", "上位セグメント", "最終同期"
- プロダクト画面では、理想を語るヒーロー文、比喩的表現、キャンペーン調の文、経営サマリー風バナーを安易に置かない
- 補足文は、対象範囲、挙動、鮮度、判断材料としての価値を一文で説明する
- ホームページのヒーローや広告にも載りそうな文なら、より業務UIらしい表現へ書き直す
- 運用、監視、判断の助けにならないセクションは削る

見出し、ラベル、数値だけを流し見しても、すぐにページの意味が分かる状態を目指します。

## モーション

モーションは、ノイズではなく、存在感と情報の強弱を作るために使います。

ビジュアル主導のUIでは、少なくとも 2〜3 個の意図あるモーションを入れてください。

- ヒーロー内の導入アニメーションを1つ
- スクロール連動、sticky、または奥行きを感じる演出を1つ
- hover、reveal、layout transition のいずれかで操作感を明確にするものを1つ

使えるなら Framer Motion を優先します。特に次の用途に向いています。

- セクションの出現演出
- shared layout transitions
- スクロール連動の opacity / translate / scale の変化
- sticky を使ったストーリーテリング
- 物語性のあるカルーセル
- メニュー、ドロワー、モーダルの存在感づくり

モーションの条件:

- 短い録画でも分かること
- モバイルでも滑らかなこと
- 速く、節度があること
- ページ全体で一貫していること
- 装飾にしかなっていないなら削除すること

## 厳守ルール

- デフォルトでカードを使わない
- デフォルトでヒーローカードを使わない
- brief が full bleed を求めているなら、囲み型や中央寄せヒーローにしない
- 1セクションに主役のアイデアは1つまで
- セクションを理解させるために、小さなUI部品を大量投入しない
- ブランドページでは、見出しがブランド名より強くなりすぎない
- 水増しのコピーを書かない
- テキストが落ち着いた片側に自然に乗らない限り、安易に split-screen
  hero にしない
- 明確な理由がない限り、書体は2種類まで
- 既存プロダクトに明確なルールがない限り、アクセントカラーは1色まで

## 失敗例として扱うもの

- 第一印象が、よくある SaaS のカードグリッド
- 画像は良いのにブランドの存在感が弱い
- 強い見出しはあるのに、取るべき行動が分からない
- 文字の背後に情報量の多い画像がある
- 同じ雰囲気の話を繰り返すセクション
- 意味のないカルーセル
- レイアウトではなく、積み重ねたカードだけで作られたアプリUI

## 最終チェック

- 最初の画面で、ブランドまたはプロダクトが明確に伝わるか
- 強いビジュアルアンカーが1つあるか
- 見出しだけ見てもページの意味が分かるか
- 各セクションに役割が1つずつあるか
- カードは本当に必要か
- モーションは情報の強弱や空気感の改善に効いているか
- 装飾的なシャドウをすべて消しても、上質さが残るか
